Tanarea SDK - Zarządzanie plikami
Zarządzanie plikami jest intuicyjne – można je tworzyć i usuwać. Natomiast czy można to samo powiedzieć o interfejsie dostarczanym przez standard C++? Niekoniecznie.
Z tego powodu Tanarea SDK implementuje swój własny interfejs do zarządzania plikami, a Ty dowiesz się jak z niego korzystać.
Zacznijmy od sprawdzania informacji o pliku, na przykład czy w ogóle istnieje.
#include <iostream>
#include <TanareaSDK.hpp>

int main()
{
tnr::file::tnrFileInfoSet FileInfo("file.txt");

if(FileInfo.IsFileExist)
{
std::cout << FileInfo.FileName << " exist" << std::endl;
std::cout << "File size: " << FileInfo.FileSize << std::endl;
}
return 0;
 }
W powyższym przykładzie najpierw utworzyłem obiekt klasy tnr::file::tnrFileInfoSet podając jednocześnie do konstruktora ścieżkę do pliku do sprawdzenia.
Użyta klasa podczas konstruowania zbiera wszystkie informacje nt. pliku. Następnie w bloku if sprawdzam czy plik istnieje, jeśli tak to wyświetlam nazwę pliku oraz jego rozmiar.

Skoro wiadomo już, że plik istnieje, to można spróbować go otworzyć. Mamy do wyboru dwa sposoby:
- Za pomocą klasy tnr::file::tnrFileText
- Za pomocą klasy tnr::file::tnrFileBinary

Pierwszy służy do zarządzania plikami tekstowymi, drugi do plików binarnych. Z racji, że nazwa naszego pliku ma rozszerzenie .txt wybór jest oczywisty.
#include <iostream>
#include <TanareaSDK.hpp>

int main()
{
// FIRST SEGMENT
 {
tnr::file::tnrFileText File("file.txt", tnr::file::tnrFileOpenMode::READ);

while (!File.IsEndOfFile())
{
std::cout << File.ReadLine() << std::endl;
}
}
// SECOND SEGMENT
 {
tnr::file::tnrFileText File("file.txt", tnr::file::tnrFileOpenMode::READ);

while (!File.IsEndOfFile())
{
std::cout << File.ReadFor("#") << std::endl;
}
}
// THIRD SEGMENT
 {
tnr::file::tnrFileText File("file.txt", tnr::file::tnrFileOpenMode::READ);

while (!File.IsEndOfFile())
{
std::cout << File.Read() << std::endl;
}
}
return 0;
 }
Kod podzieliłem na segmenty zależnie od użytego sposobu odczytywania danych z pliku. Podstawową częścią każdego segmentu jest utworzenie obiektu klasu tnrFileText.
Podałem do konstruktora nazwę pliku do otwarcia, oraz tryb w jakim plik ma być otwarty. Chcemy przejrzeć tylko zawartość pliku w związku z tym wystarczy flaga READ.

Idąc dalej każdy segment zawiera również pętlę while bazującą na wartości zwracanej przez metodę IsEndOfFile(). Metoda ta zwraca informację czy dotarliśmy do końca pliku.
Pętla więc trwa dopóki nie odczytaliśmy całego pliku.

Przejdźmy do samych segmentów. W pierwszym segmencie wywoływana jest metoda ReadLine(). Oznacza to, że odczytujemy plik po jednej linijce danych
(czyli od początku linii do białego znaku nowej linii \n). Dane zwracane przez ReadLine() nie zawierają znaku nowej linii.

Drugi segment odczytuje plik w podobny sposób co za pomocą ReadLine - z tą różnicą, że ReadFor odczytuje dane aż napotka znak podany w argumencie
(w podanym przykładzie odczytuje od początku danych do znaku #).

Z kolei trzeci segment odczytuje zwyczajnie znak po znaku. Wyświetlane więc w tym przykładzie są pojedyncze znaki.

Co z zapisywaniem do plików tekstowych? Wystarczy dostosować flagi:
#include <iostream>
#include <TanareaSDK.hpp>

int main()
{
tnr::file::tnrFileText File
(
"innyplik.txt" ,
static_cast<tnr::file::tnrFileOpenMode>
(
tnr::file::tnrFileOpenMode::WRITE |
tnr::file::tnrFileOpenMode::ALLOW_TRUNCATE
)
);
File.Write("Kot jednak ma Ale");


return 0;
 }
Zmiana flagi READ na WRITE sprawiła, że teraz plik jest otwarty w trybie do zapisu. Dodatkowo zdefiniowałem również flagę ALLOW_TRUNCATE, która powoduje wyczyszczenie obecnej zawartości pliku,
by zapisywać do pustego pliku. Pamiętaj, że interfejs Tanarea SDK w przeciwieństwie do standardowego interfejsu C++ nie pozwala na jednoczesny zapis i odczyt z/do pliku.
Musisz być zdecydowany co chcesz z plikiem robić!

Samo zapisywanie to banał – wystarczy wywołać metodę Write, która jako argument przyjmuje tekst do zapisu. To tyle jeśli chodzi o obsługę plików tekstowych.
Przejdźmy więc do plików binarnych.

Tutaj sytuacja wygląda zgoła inaczej. Zacznijmy od odczytu:
#include <iostream>
#include <TanareaSDK.hpp>

int main()
{
// FIRST SEGMENT
 {
struct MyData
{
tnr::ui32 FirstInt;
tnr::f16 HalfsArray[3];
};
tnr::file::tnrFileBinary File(„file.bin”, tnr::file::tnrFileOpenMode::READ);
MyData Data = File.ReadBinary<MyData>();
}
// SECOND SEGMENT
 {
struct MyData
{
tnr::ui32 FirstInt;
tnr::f16 HalfsArray[3];
};
tnr::file::tnrFileBinary File(„file.bin”, tnr::file::tnrFileOpenMode::READ);
std::vector<char> Data = File.ReadBinary(sizeof(MyData));
MyData FinalData = *reinterpret_cast<MyData*>(Data.data());
}
return 0;
 }
W każdym segmencie początek jest ten sam – tworzymy obiekt (tym razem klasy tnrFileBinary), podajemy ścieżkę do pliku oraz flagi trybu otwarcia. Inne są jednak sposoby odczytu.

Pierwszy segment reprezentuje „odczyt typowany”. To znaczy – metoda ReadBinary<T> celowo ma przeciążenie, które jest szablonem.
Pozwala to na odczytanie danych z pliku binarnego podając sam typ danych do odczytania. Przeciążenie to odczytuje n bajtów gdzie n = sizeof(T)
oraz automatycznie je konwertuje do pożądanego typu danych i zwraca w tym typie danych.

Drugi segment pokazuje konwencjonalne odczytywanie bajtów z pliku binarnego. Skorzystałem tu z innego przeciążenia ReadBinary,
gdzie metoda przyjmuje tylko ilość bajtów do odczytu i zwraca wspomniane bajty w postaci tablicy dynamicznej bajtów – std::vector.
Następnie za pomocą reinterpret_cast przekopiowałem dane z tablicy bajtów do zmiennej FinalData o docelowym typie.

Podobnie sytuacja wygląda przy zapisie do pliku binarnego:
#include <iostream>
#include <TanareaSDK.hpp>

int main()
{
// FIRST SEGMENT
 {
struct MyData
{
tnr::ui32 FirstInt;
tnr::f16 HalfsArray[3];
} Data;
tnr::file::tnrFileBinary File(„file.bin”, tnr::file::tnrFileOpenMode::WRITE);
File.WriteBinary(Data);
// SECOND SEGMENT
 {
struct MyData
{
tnr::ui32 FirstInt;
tnr::f16 HalfsArray[3];
} Data;

std::vector<char> InputBuffer(sizeof(Data));
std::memcpy(InputBuffer.data(), &Data, sizeof(Data));

tnr::file::tnrFileBinary File(„file.bin”, tnr::file::tnrFileOpenMode::WRITE);
File.WriteBinary(InputBuffer);
}
return 0;
 }
Znów – początek każdego segmentu jest już znajomy, zmieniła się tylko flaga na WRITE.

W pierwszym segmencie widać zapis bezpośrednio z danego pola prosto do pliku. Podobnie jak przy ReadBinary, WriteBinary ma przeciążenie jako szablon,
pozwala to przez dedukcję typu podawanego parametru bezproblemowo zapisywać dane bez podawania niczego więcej oprócz samych danych!

Drugi segment pokazuje zapis danych jako zwykłej tablicy bajtów. Na początku stworzyłem InputBuffer by do niego przekopiować dane,
by móc je później podać za pomocą przyjmowanego w argumencie WriteBinary std::span<char>. To przeciążenie metody WriteBinary zapisuje całą zawartość przekazaną za pomocą std::span.
